# NsCDE-Wayland Nix Bootstrap

`NsCDE-Wayland/` now has its own project root bootstrap for the extracted
`Wayland` runtime path.

Current scope:

- builds a standalone-owned `nscde_labwc` launcher/session closure from the
  extracted tree
- builds the patched bundled `labwc` compositor from `./labwc`
- builds the extracted `Haskell` runtime as `nscde-runtime`
- builds the extracted native `Wayland` clients as:
  - `nscde-paneld`
  - `nscde-pagerd`
  - `nscde-toplevel`
  - combined `nscde-wayland-clients`
- exposes a combined `nscde-wayland-bootstrap` default package with the current
  buildable launcher, session wrapper, runtime, native clients, and extracted
  assets
- materializes the reference static panel/session contracts from `nix/modules/`
- provides a local dev shell with the toolchains needed for the extracted
  `Haskell`, `C`, Python, and shell-era migration pieces
- exposes a runnable runtime verification path with `nix flake check` or
  `nix run .#runtime-check`
- exposes a runnable launcher verification path with `nix run .#launcher-check`

## Commands

Enter the standalone shell:

```sh
nix develop
```

Build the extracted runtime:

```sh
nix build .#nscde-runtime
```

Build the bundled patched compositor:

```sh
nix build .#labwc
```

Build the extracted native clients:

```sh
nix build .#nscde-wayland-clients
```

Build the current standalone bootstrap bundle:

```sh
nix build .
```

Run the packaged launcher prepare-only verification:

```sh
nix run .#launcher-check
```

Run the extracted runtime verification:

```sh
nix run .#runtime-check
```

Or as a flake check:

```sh
nix flake check
```

## Bootstrap policy

This flake now carries its own direct inputs and local `flake.lock`.

That keeps the split honest:

- `NsCDE-Wayland/` can be evaluated as its own project root
- the bootstrap no longer depends on parent-workspace path wiring
- the current lock values are aligned with the workspace baseline that the
  extraction started from

It is still a bootstrap because the launcher/session package still depends on
`legacy-shims/` for menu/theme/keybind/style glue. Those remaining shell-era
pieces still ship as `.in` templates because they need configure-style path
substitution inside the standalone package build. The new tree now owns a
packaged launcher closure, but not yet a fully de-legacyfied runtime.

One concrete seam is now gone: `menu.xml` generation is owned by
`nscde-runtime`, and the packaged bootstrap no longer ships
`nscde_labwc_menugen`.

The standalone repo now carries `labwc/` as a git submodule and the flake
builds the packaged launcher against that bundled patched compositor by
default. Initialize it with:

```sh
git submodule update --init --recursive
```

## Ownership boundary

The following root-level files are standalone-owned and are not regenerated by
`tools/sync-from-nscde.sh`:

- `flake.nix`
- `README.nix.md`
- `cabal.project`
- `nscde-wayland-runtime.cabal`
- `tools/check-launcher.sh`
- `tools/check-runtime.sh`

The extracted source trees under `haskell/`, `nix/`, `src/`, `assets/`, and
selected integration trees remain sync-managed migration inputs.
