# NsCDE-Wayland

This directory is the standalone repository root for the `Wayland` runtime.

It lives beside the original [NsCDE](/home/neuron/Projects/CDEwrapper/NsCDE)
tree on purpose:

- `NsCDE/` remains the legacy upstream, behavior reference, and imported-asset
  source
- `NsCDE-Wayland/` is the new runtime ownership target for:
  - `Haskell` semantic runtime code
  - `C` `Wayland` clients
  - `Nix` static configuration and wrapper material
  - extracted assets that the `Wayland` path still consumes

This repo is the runtime owner now. It still carries some transitional imported
inputs and sync tooling, but the `Wayland` adaptation should no longer be
treated as a patch layered onto the original `NsCDE/` tree.

It now also has a local bootstrap build root:

- `flake.nix`
  - standalone dev/build entrypoint for the extracted runtime and native
    `Wayland` clients
- `nscde-wayland-runtime.cabal`
  - root `Cabal` package for the extracted `Haskell` runtime
- `tools/check-runtime.sh`
  - runnable verification for the first extracted runtime contracts
- `tools/check-launcher.sh`
  - runnable prepare-only verification for the packaged standalone launcher
    and session closure

Current verified bootstrap status:

- `nix flake check path:/.../NsCDE-Wayland` passes
- `nix run path:/.../NsCDE-Wayland#launcher-check` passes
- the default package now produces a runnable `nscde_labwc` bootstrap bundle
  with the extracted runtime, native daemons, assets, and temporary shell-era
  shims wired together
  - the launcher now defaults to the bundled patched `labwc/` submodule build

## Layout

- `haskell/`
  - extracted runtime modules and executable entrypoints
- `src/`
  - extracted native `Wayland` clients and shared native support code
  - now includes the currently used `nscde_common/nscde-pixel-icon.[ch]`
    helper that `nscde_paneld` still depends on
- `nix/`
  - extracted static profile/default modules
- `assets/`
  - extracted palettes, fontsets, backdrops, photos, defaults, and icon assets
  - includes the current `fvwm/` runtime reference files still consumed by the
    launcher path (`Font-*.fvwmconf`, `Keybindings.*`)
- `xdg/`
  - extracted desktop/session integration assets
- `labwc/`
  - standalone-owned patched compositor source as a git submodule
  - provides the SSD border and titlebar rendering path used by the standalone
    launcher package
- `legacy-shims/`
  - temporary imported shell-era helpers still needed while the runtime split
    is incomplete
  - the remaining `.in` files live here because these pieces still use
    configure-style substitution and have not yet been fully moved into
    `Haskell`, `Nix`, or native clients
  - `nscde_labwc_menugen` is no longer on the packaged launcher path; `menu.xml`
    is now published by `nscde-runtime`
  - `nscde_labwc_keybindgen` is now a compatibility wrapper; the launcher
    prefers `nscde-runtime` for keybind generation too
- `doc/`
  - standalone-split notes and asset provenance
- `tools/sync-from-nscde.sh`
  - refreshes selected imported migration inputs from the current `NsCDE/`
    source tree when intentionally needed

## Current policy

This tree is the owner of the `Wayland` runtime path.

The old `NsCDE/` tree should be treated as:

- reference implementation for preserved behavior
- source of legacy assets worth retaining
- migration input, not long-term runtime ownership

## Sync

Refresh selected imported migration inputs from the current `NsCDE/` tree
with:

```sh
./tools/sync-from-nscde.sh
```

Standalone-owned bootstrap files live at the repo root and are intentionally
not regenerated by the sync script.

Quick standalone build commands:

```sh
git submodule update --init --recursive
nix build .#labwc
nix build .
nix build .#nscde-wayland-clients
nix run .#launcher-check
nix run .#runtime-check
```
